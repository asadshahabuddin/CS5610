Session

Description

Until last week, a user who signed in would not have the ability to sign out of the application. The issue has been addressed this time around. A session can be terminated any time by clicking on the Sign out button present on the navigation bar.

Sign in using your previously created credentials and sign out. You can't access any of the pages once you have.

Node.js server hosted at OpenShift

Click here to go the experiment.

Documentation

Sign out

After you signed out, try accessing any of the pages by editing the URL. The pages won't be accessible any longer. If you open your browser's console window, you'll notice appropriate logging.

/* Server */
app.post("/api/logout", function(req, res)
{
req.logOut();
res.send(200);
});

/* Client */
$scope.logout = function()
{
if($rootScope
&& typeof $rootScope.currentUser != "undefined")
{
$http.post("/api/logout")
.success(function()
{
console.log("%c [echo] Logged out user '" + $rootScope.currentUser.username + "'",
"font-family: Courier New;");
});
}
$location.url("/");
};
