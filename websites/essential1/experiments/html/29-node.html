Cookie

Description

Browser cookies are exhibited as part of this experiment. You need to fill up the page with all the information it needs. After that, you click on the Sing in button. This will set a cookie which can be inspected using your browser's console window. An alert message will tell you the same.

Documentation

Browser Sessions

Cookies are set to enable stateful sessions. They are used by services or websites to track a person as long as they stay signed in or until the cookie expires. When you click the login button after providing enough information, an AngularJS handler sends a POST request to the server at a link which is handled by the Node.js listener shown below. This listener responds with a cookie and an alert pops up after validating success.

/* Post listeners : BEGIN */
app.post("/api/cookie", function(req, res)
{
res.cookie("username", req.body.fname);
res.writeHead(200, {"Content-Type":"text/html"});
res.write("Cookie Set. You can inspect it by typing 'document.cookie' " +
"in your browser's console.");
res.end();
});
