Guest

Description

My project website required the user to sign in before being able to use any of its features. That restriction has been relaxed and one can sign in as a guest now. The profile page automatically unlocks relevant features and keeps the privileged features restricted. Guest profile is also cutomized to show the name Guest on the navigation bar as his/her profile link and initials switch to GU (Guest User). To login as a guest, click on the Try for free button next to the Sign in button.

Node.js server hosted at OpenShift

Additional routes have been created to faciliate the guest feature. If I were to unlock some of the pages through existing routes, the user would not be re-authenticated on page refresh. That would have resulted in a potential vulnerability. Every page to be accessed by the guest is never passed through the authentication filter and creates a clear separation.

app.config(function($routeProvider)
{
$routeProvider
.when("/",
{
templateUrl: "html/welcome.html",
controller: "LoginCtrl"
})
.when("/profile",
{
templateUrl: "html/profile.html",
controller: "ProfileCtrl",
resolve: {
loggedin: checkLogin
}
})
.when("/profileg",
{
templateUrl: "html/profile.html",
controller: "ProfileCtrl"
})
.when("/news",
{
templateUrl: "html/news.html",
controller: "NewsCtrl",
resolve: {
loggedin: checkLogin
}
})
.when("/newsg",
{
templateUrl: "html/news.html",
controller: "NewsCtrl"
})
.when("/books",
{
templateUrl: "html/books.html",
controller: "BooksCtrl",
resolve: {
loggedin: checkLogin
}
})
.when("/booksg",
{
templateUrl: "html/books.html",
controller: "BooksCtrl"
})
.when("/music",
{
templateUrl: "html/music.html",
controller: "MusicCtrl",
resolve: {
loggedin: checkLogin
}
})
.when("/musicg",
{
templateUrl: "html/music.html",
controller: "MusicCtrl"
})
.otherwise(
{
redirectTo: "/"
});
